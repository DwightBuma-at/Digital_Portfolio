Complaint Management System

Django Python Supabase PostgreSQL REST Framework SendGrid Railway
View on GitHub Visit Live Site

The Problem

Barangays in Davao City lacked a centralized digital system for residents to submit and track complaints. The existing process relied heavily on manual paper-based forms, walk-in submissions, and phone calls, leading to:

  • Long response times and poor tracking of complaint status
  • Lack of transparency for residents on the progress of their concerns
  • Inefficient communication between residents and barangay officials
  • Difficulty in generating reports and analyzing complaint trends

The Solution

I architected and deployed a full-stack web application that digitizes the entire complaint management workflow. The system allows residents to submit complaints online with photo evidence, while barangay officials can efficiently manage, track, and resolve issues through a dedicated admin dashboard.

My Role & Key Contributions

  • Backend Architecture: Built the entire backend using Django and Django REST Framework, designing RESTful API endpoints for complaint submission, status updates, and admin operations
  • Database Design: Implemented PostgreSQL database using Supabase, designing normalized schemas for users, complaints, barangay data, and audit logs with proper indexing for performance
  • Authentication & Authorization: Developed role-based access control (RBAC) system with three user types: residents, barangay staff, and super admins, each with specific permissions
  • Real-Time Notifications: Integrated SendGrid email service to send automated notifications when complaints are submitted, updated, or resolved
  • Deployment & DevOps: Deployed the application on Railway with continuous deployment from GitHub, configured environment variables, and set up SSL certificates

Technical Decisions & Why

  • Django + Supabase: Chose Django for rapid development with built-in admin interface and ORM. Supabase provided managed PostgreSQL with real-time capabilities and easier scaling compared to traditional MySQL setups
  • Railway over Heroku: Selected Railway for deployment due to better pricing, faster deployment times, and seamless GitHub integration without complex buildpacks
  • SendGrid for Emails: Implemented SendGrid instead of SMTP for reliable email delivery with built-in analytics and reputation management
  • REST Framework: Used DRF to create a clean API structure that could potentially support mobile app integration in the future

Challenges & Solutions

  • Challenge: Handling image uploads from residents reporting complaints
    Solution: Implemented Cloudinary integration for cloud-based image storage and optimization, reducing server load and ensuring fast image delivery
  • Challenge: Ensuring email deliverability in production environment
    Solution: Configured SPF and DKIM records with SendGrid, implemented email verification, and added fallback notification system
  • Challenge: Managing different barangay jurisdictions and permissions
    Solution: Created a hierarchical permission system where each barangay admin can only view/manage complaints from their specific barangay

Results & Impact

✓ Deployed and serving 100+ barangay officials and residents

✓ Reduced manual communication overhead by 70% through automated email notifications

✓ Average complaint resolution time improved with better tracking and accountability

✓ Successfully handling peak loads during community events with zero downtime

What I Learned

This project taught me the importance of user-centered design when building government-facing applications. I learned to balance simplicity for non-technical users with robust functionality for administrators. The experience of deploying and maintaining a production application with real users gave me invaluable insights into error handling, logging, and the importance of comprehensive testing before releases.

The Problem

A local art business shop in Davao City needed a comprehensive e-commerce solution but was operating with disconnected systems: spreadsheets for online orders, manual POS, Facebook messages for bookings, and paper-based purchasing. This fragmentation caused inventory discrepancies, delayed responses, and no unified business insights.

The Solution

I architected and developed an integrated e-commerce platform as a System Integration and Architecture project, unifying all business operations into a single system with 5 modules: Ordering, POS, Booking, Purchasing, and Inventory Management. The platform provides real-time synchronization and centralized analytics.

My Role & Key Contributions

  • Project Planning & Architecture: Led system design from requirements gathering through technical specifications, creating architecture diagrams and module integration plans
  • Full-Stack Development: Built entire application end-to-end—designed normalized PostgreSQL database schemas, developed Django REST APIs for all modules, created responsive Bootstrap frontend with JavaScript
  • System Integration: Integrated all 5 modules with real-time data synchronization using database triggers and signals, ensuring inventory accuracy across POS and online orders
  • Deployment & DevOps: Configured production environment, set up database migrations, implemented backup strategies, and deployed live application
  • Documentation: Created comprehensive technical documentation including system architecture, API endpoints, database schemas, and user manuals

Technical Decisions & Why

  • Django Monolith: Chose a monolithic Django architecture over microservices for faster development and easier state management given the team size and timeline
  • PostgreSQL Triggers: Implemented database triggers to automatically update inventory levels when orders are placed or POS transactions occur, ensuring real-time accuracy
  • Session-based Auth: Used Django's built-in session authentication instead of JWT for simpler implementation and better security for a primarily web-based application
  • Bootstrap for UI: Leveraged Bootstrap for rapid UI development while maintaining responsive design across devices

Challenges & Solutions

  • Challenge: Preventing race conditions when multiple staff process orders simultaneously
    Solution: Implemented database-level locks and transaction isolation to ensure inventory updates are atomic
  • Challenge: Coordinating work across 4 developers with different skill levels
    Solution: Established clear module boundaries, created comprehensive documentation, and held daily standups to address blockers
  • Challenge: Migrating existing product data from spreadsheets
    Solution: Created custom Django management commands to bulk import and validate data with error reporting

Results & Impact

✓ Successfully delivered complete System Integration & Architecture project from planning to deployment

✓ Unified 5 disconnected business operations into single integrated e-commerce platform

✓ Eliminated inventory discrepancies and reduced booking response time from hours to minutes

✓ Empowered local Davao City business with professional e-commerce solution and real-time analytics

What I Learned

Leading this System Integration project end-to-end taught me the full software development lifecycle—from stakeholder meetings and requirements analysis to architecture design, implementation, deployment, and documentation. I gained deep understanding of how to integrate multiple business systems, the importance of comprehensive planning before coding, and how proper documentation ensures long-term maintainability. Most importantly, I learned to deliver solutions that directly solve real business problems for local entrepreneurs.

The Problem

Filipino local juice businesses struggle to establish an online presence due to the complexity and cost of e-commerce platforms. Many rely solely on social media for orders, which:

  • Lacks professional branding and credibility
  • Makes it difficult for customers to browse full product catalogs
  • Provides no structured ordering or cart management system
  • Limits payment options and delivery tracking capabilities

The Solution

I created a lightweight, fully responsive e-commerce website that runs entirely in the browser without requiring backend infrastructure. The solution provides a professional shopping experience with product catalog, cart management, and checkout—all deployable on free hosting platforms like GitHub Pages.

My Role & Key Contributions

  • Frontend Development: Designed and built the entire user interface from scratch using HTML5, CSS3, and vanilla JavaScript without relying on heavy frameworks
  • Shopping Cart Logic: Implemented a complete shopping cart system with add/remove items, quantity controls, and real-time price calculations using JavaScript
  • State Management: Utilized the Browser Local Storage API to persist cart data across sessions, ensuring customers don't lose their selections on page refresh
  • Responsive Design: Created mobile-first responsive layouts with Bootstrap 5, ensuring optimal viewing experience on phones, tablets, and desktops
  • User Experience: Added smooth animations, transitions, and interactive feedback to enhance the overall shopping experience

Technical Decisions & Why

  • Vanilla JavaScript: Chose vanilla JS over React/Vue to keep the project lightweight, fast-loading, and easy to customize for non-technical business owners
  • Local Storage for Cart: Used Local Storage instead of cookies for larger storage capacity and simpler implementation without needing server-side sessions
  • Bootstrap 5: Leveraged Bootstrap for rapid development of professional-looking components while maintaining small bundle sizes
  • Static Site Deployment: Designed as a static site to enable free hosting on GitHub Pages, making it accessible to small businesses with limited budgets

Challenges & Solutions

  • Challenge: Managing cart state without a backend database
    Solution: Implemented a robust Local Storage system with JSON serialization, data validation, and error handling for edge cases
  • Challenge: Handling checkout without payment gateway integration
    Solution: Created a multi-option checkout form supporting COD, GCash, and bank transfer with clear instructions, allowing business owners to handle payments manually
  • Challenge: Optimizing performance without build tools
    Solution: Minimized DOM manipulations, used CSS for animations instead of JS, and implemented lazy loading for images

Results & Impact

✓ 100% client-side application with zero backend costs

✓ Fully responsive design working seamlessly across all devices

✓ Complete e-commerce functionality without database requirements

✓ Free deployment on GitHub Pages, making it accessible to small businesses

What I Learned

This project reinforced the value of understanding web fundamentals. By building without frameworks, I gained deeper knowledge of browser APIs, event handling, and state management. I also learned to design solutions that match business constraints—sometimes a simple static site is more appropriate than a complex full-stack application. The experience taught me to prioritize user experience and performance even without modern build tools.

ADDU Fitness Management System

The Problem

The Ateneo de Davao University Fitness and Wellness Center was facing significant operational challenges with their traditional paper-based and Excel spreadsheet system:

  • Front-desk staff struggled to track member check-ins efficiently using manual logbooks and Excel files
  • No real-time visibility into membership status (active vs. expired), leading to awkward situations at check-in
  • Difficult to generate reports on gym usage, member trends, and facility utilization
  • Time-consuming membership renewal process with manual record updates across multiple files
  • Risk of data loss due to lack of proper backup systems for Excel files

The Solution

I developed a comprehensive web-based gym management system that modernizes the entire membership and check-in workflow. The system provides front-desk staff with an intuitive interface for managing member registrations, processing check-ins, tracking membership status, and generating usage reports—all accessible from any device with a web browser. The solution replaces Excel spreadsheets with a proper database structure while maintaining simplicity and ease of use.

My Role & Key Contributions

  • System Design & Architecture: Designed the complete system architecture including data models for members, check-ins, memberships, and staff authentication using LocalStorage as the database
  • Member Management System: Built comprehensive member registration with detailed profile information, membership type selection, and automated expiration tracking
  • Real-Time Check-In System: Implemented a smart check-in dashboard with member search, instant status validation (active/expired), and timestamp logging with staff accountability
  • Data Analytics Dashboard: Created statistics views showing daily check-ins, active memberships, and member trends to help administrators make informed decisions
  • User Interface Design: Designed a clean, professional interface using Ateneo's blue color scheme with WordPress-inspired modern design patterns for easy adoption by non-technical staff
  • Responsive Layout: Ensured the system works seamlessly on desktop computers at the front desk and mobile devices for on-the-go access

Technical Decisions & Why

  • LocalStorage Database: Chose browser LocalStorage over backend database for zero infrastructure costs, instant deployment, and simplified maintenance—perfect for a university fitness center with budget constraints
  • Vanilla JavaScript: Used pure JavaScript ES6+ instead of React/Vue to keep the application lightweight, fast-loading, and easy for future IT staff to maintain without framework dependencies
  • Client-Side Authentication: Implemented simple but effective staff login system suitable for the single-location use case without overengineering security
  • Bootstrap 5 Framework: Leveraged Bootstrap's proven components and grid system for rapid development of professional, accessible UI
  • GitHub Pages Deployment: Deployed on GitHub Pages for free, reliable hosting with HTTPS, making the system accessible 24/7 without ongoing server costs

Challenges & Solutions

  • Challenge: Managing data persistence without a backend database
    Solution: Designed a robust LocalStorage-based data layer with proper JSON serialization, data validation, and import/export capabilities for backup
  • Challenge: Preventing duplicate check-ins for the same member on the same day
    Solution: Implemented timestamp-based duplicate detection logic that checks existing records before allowing new check-ins
  • Challenge: Making membership expiration tracking automatic and reliable
    Solution: Built date calculation logic that automatically flags expired memberships with visual indicators (red badges) on the dashboard
  • Challenge: Ensuring front-desk staff could search members quickly during busy hours
    Solution: Created case-insensitive search functionality that matches partial names, student IDs, or email addresses with instant filtering

Results & Impact

✓ Modernized Ateneo de Davao University Fitness Center's operations by replacing outdated Excel-based system

✓ Reduced check-in time from minutes to seconds with instant member lookup and status validation

✓ Eliminated membership tracking errors and awkward expired-membership situations at front desk

✓ Provided administrators with real-time analytics on gym usage and member trends for better facility planning

✓ Zero infrastructure costs with free GitHub Pages hosting—significant savings for university budget

What I Learned

This project taught me the importance of understanding the real needs of non-technical users. By spending time with the front-desk staff to understand their daily workflows, I could design a system that genuinely improved their efficiency rather than adding complexity. I learned that sometimes the best solution isn't the most technically sophisticated one—using LocalStorage instead of a backend database made the system more maintainable and cost-effective for the university. The experience reinforced that good software solves real problems for real people, and success is measured by user adoption, not by technical complexity.

Elevation Fitness Gym Management System

The Problem

Elevation Fitness Gym in Davao City was facing operational inefficiencies with their outdated manual logbook system for tracking member attendance and managing memberships:

  • Staff relied on handwritten logbooks to record member check-ins, making it difficult to track attendance patterns and verify membership status
  • No automated way to identify expired memberships, leading to awkward confrontations and revenue loss from unrenewed members
  • Manual record-keeping made it challenging to generate reports on gym usage and member activity
  • Membership renewal process required staff to manually search through paper records and update multiple documents
  • No central database meant risk of lost records and inability to quickly look up member information
  • Difficulty tracking trainer assignments and schedules

The Solution

I developed a comprehensive web-based gym management system that completely replaces the manual logbook system with a modern digital platform. The solution provides staff with real-time member management, automated attendance tracking, instant membership status validation, and streamlined renewal workflows—all accessible from any device with a web browser. The system centralizes all gym operations in one intuitive interface designed specifically for Elevation Fitness's needs.

My Role & Key Contributions

  • Complete System Architecture: Designed and implemented the entire system architecture with separate modules for dashboard, members, attendance, trainers, and renewals using LocalStorage as the persistent database
  • Member Management Module: Built comprehensive member directory with full CRUD operations, photo uploads, emergency contacts, multiple membership plan options (Daily, Weekly, Monthly, 3/6/12 months), and automated expiry tracking
  • Attendance Tracking System: Developed real-time check-in functionality with member search, duplicate prevention (same day), expired member blocking, and complete attendance history with timestamp logging
  • Renewal Workflow: Created dedicated renewal interface that displays expired members, auto-fills renewal forms, calculates new expiry dates, and tracks payment status (Paid/Pending/Partial)
  • Dashboard Analytics: Implemented statistics dashboard showing total members, active vs. expired breakdown, and today's check-in counts for quick operational overview
  • User Interface Design: Designed modern, clean interface with custom CSS variables, responsive layouts, and professional color scheme optimized for gym staff workflow

Technical Decisions & Why

  • LocalStorage Database: Selected browser LocalStorage over backend database to eliminate infrastructure costs, enable instant deployment, and simplify maintenance—ideal for a small local gym with limited IT budget
  • Vanilla JavaScript ES6+: Used pure JavaScript instead of React/Vue to keep the application lightweight, fast, and maintainable by future staff without requiring framework expertise
  • Modular Page Architecture: Separated functionality into distinct HTML pages (index, members, attendance, trainers) for clear navigation and easier feature updates
  • Bootstrap 5 + Custom CSS: Combined Bootstrap's responsive grid with custom CSS variables for consistent branding while maintaining rapid development speed
  • GitHub Pages Hosting: Deployed on GitHub Pages for zero hosting costs, automatic HTTPS, and reliable 24/7 availability without server management
  • Font Awesome Icons: Integrated Font Awesome for professional iconography that enhances user experience and visual clarity

Challenges & Solutions

  • Challenge: Ensuring data persistence and preventing data loss without a backend database
    Solution: Implemented robust LocalStorage management with JSON serialization, validation checks, and defensive programming to handle browser storage limits and edge cases
  • Challenge: Preventing duplicate same-day check-ins while maintaining fast lookup performance
    Solution: Created efficient timestamp-based duplicate detection that checks today's date against existing check-in records before allowing new entries
  • Challenge: Automatically flagging expired memberships without manual staff checks
    Solution: Built date comparison logic that runs on page load, calculates days remaining, and displays visual status indicators (Active/Expiring Soon/Expired) with color-coded badges
  • Challenge: Making member search fast enough for peak gym hours
    Solution: Implemented real-time case-insensitive search that filters across name, ID, email, and contact fields with debounced input for optimal performance
  • Challenge: Streamlining the renewal process to reduce staff workload
    Solution: Designed one-click renewal workflow where selecting an expired member auto-fills the form, and choosing a plan automatically calculates the new expiry date

Results & Impact

✓ Completely modernized Elevation Fitness Gym's operations by replacing manual logbook with digital platform

✓ Reduced member check-in time from 2-3 minutes to under 10 seconds with instant member lookup and validation

✓ Eliminated expired membership issues with automatic status tracking and visual alerts

✓ Streamlined renewal process from 15+ minutes to under 2 minutes per member

✓ Enabled data-driven decisions with real-time attendance analytics and member trend reporting

✓ Zero ongoing costs with free GitHub Pages hosting—saving gym hundreds in software licensing fees

What I Learned

Building this system for Elevation Fitness taught me the critical importance of designing software around actual user workflows rather than technical capabilities. By observing how gym staff interacted with their logbook system and understanding their pain points, I could create a digital solution that felt natural and immediately valuable rather than forcing them to adapt to unfamiliar processes. I learned that sometimes the best technology choice is the simplest one that solves the problem—LocalStorage was perfect for this use case despite being "less sophisticated" than a backend database. Most importantly, this project reinforced that great software empowers users to do their jobs better, and success is measured by how quickly staff adopt the system and how much time it saves them daily.
