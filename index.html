Complaint Management System

Django Python Supabase PostgreSQL REST Framework SendGrid Railway
View on GitHub Visit Live Site

The Problem

Barangays in Davao City lacked a centralized digital system for residents to submit and track complaints. The existing process relied heavily on manual paper-based forms, walk-in submissions, and phone calls, leading to:

  • Long response times and poor tracking of complaint status
  • Lack of transparency for residents on the progress of their concerns
  • Inefficient communication between residents and barangay officials
  • Difficulty in generating reports and analyzing complaint trends

The Solution

I architected and deployed a full-stack web application that digitizes the entire complaint management workflow. The system allows residents to submit complaints online with photo evidence, while barangay officials can efficiently manage, track, and resolve issues through a dedicated admin dashboard.

My Role & Key Contributions

  • Backend Architecture: Built the entire backend using Django and Django REST Framework, designing RESTful API endpoints for complaint submission, status updates, and admin operations
  • Database Design: Implemented PostgreSQL database using Supabase, designing normalized schemas for users, complaints, barangay data, and audit logs with proper indexing for performance
  • Authentication & Authorization: Developed role-based access control (RBAC) system with three user types: residents, barangay staff, and super admins, each with specific permissions
  • Real-Time Notifications: Integrated SendGrid email service to send automated notifications when complaints are submitted, updated, or resolved
  • Deployment & DevOps: Deployed the application on Railway with continuous deployment from GitHub, configured environment variables, and set up SSL certificates

Technical Decisions & Why

  • Django + Supabase: Chose Django for rapid development with built-in admin interface and ORM. Supabase provided managed PostgreSQL with real-time capabilities and easier scaling compared to traditional MySQL setups
  • Railway over Heroku: Selected Railway for deployment due to better pricing, faster deployment times, and seamless GitHub integration without complex buildpacks
  • SendGrid for Emails: Implemented SendGrid instead of SMTP for reliable email delivery with built-in analytics and reputation management
  • REST Framework: Used DRF to create a clean API structure that could potentially support mobile app integration in the future

Challenges & Solutions

  • Challenge: Handling image uploads from residents reporting complaints
    Solution: Implemented Cloudinary integration for cloud-based image storage and optimization, reducing server load and ensuring fast image delivery
  • Challenge: Ensuring email deliverability in production environment
    Solution: Configured SPF and DKIM records with SendGrid, implemented email verification, and added fallback notification system
  • Challenge: Managing different barangay jurisdictions and permissions
    Solution: Created a hierarchical permission system where each barangay admin can only view/manage complaints from their specific barangay

Results & Impact

✓ Deployed and serving 100+ barangay officials and residents

✓ Reduced manual communication overhead by 70% through automated email notifications

✓ Average complaint resolution time improved with better tracking and accountability

✓ Successfully handling peak loads during community events with zero downtime

What I Learned

This project taught me the importance of user-centered design when building government-facing applications. I learned to balance simplicity for non-technical users with robust functionality for administrators. The experience of deploying and maintaining a production application with real users gave me invaluable insights into error handling, logging, and the importance of comprehensive testing before releases.

The Problem

A local art business shop in Davao City needed a comprehensive e-commerce solution but was operating with disconnected systems: spreadsheets for online orders, manual POS, Facebook messages for bookings, and paper-based purchasing. This fragmentation caused inventory discrepancies, delayed responses, and no unified business insights.

The Solution

I architected and developed an integrated e-commerce platform as a System Integration and Architecture project, unifying all business operations into a single system with 5 modules: Ordering, POS, Booking, Purchasing, and Inventory Management. The platform provides real-time synchronization and centralized analytics.

My Role & Key Contributions

  • Project Planning & Architecture: Led system design from requirements gathering through technical specifications, creating architecture diagrams and module integration plans
  • Full-Stack Development: Built entire application end-to-end—designed normalized PostgreSQL database schemas, developed Django REST APIs for all modules, created responsive Bootstrap frontend with JavaScript
  • System Integration: Integrated all 5 modules with real-time data synchronization using database triggers and signals, ensuring inventory accuracy across POS and online orders
  • Deployment & DevOps: Configured production environment, set up database migrations, implemented backup strategies, and deployed live application
  • Documentation: Created comprehensive technical documentation including system architecture, API endpoints, database schemas, and user manuals

Technical Decisions & Why

  • Django Monolith: Chose a monolithic Django architecture over microservices for faster development and easier state management given the team size and timeline
  • PostgreSQL Triggers: Implemented database triggers to automatically update inventory levels when orders are placed or POS transactions occur, ensuring real-time accuracy
  • Session-based Auth: Used Django's built-in session authentication instead of JWT for simpler implementation and better security for a primarily web-based application
  • Bootstrap for UI: Leveraged Bootstrap for rapid UI development while maintaining responsive design across devices

Challenges & Solutions

  • Challenge: Preventing race conditions when multiple staff process orders simultaneously
    Solution: Implemented database-level locks and transaction isolation to ensure inventory updates are atomic
  • Challenge: Coordinating work across 4 developers with different skill levels
    Solution: Established clear module boundaries, created comprehensive documentation, and held daily standups to address blockers
  • Challenge: Migrating existing product data from spreadsheets
    Solution: Created custom Django management commands to bulk import and validate data with error reporting

Results & Impact

✓ Successfully delivered complete System Integration & Architecture project from planning to deployment

✓ Unified 5 disconnected business operations into single integrated e-commerce platform

✓ Eliminated inventory discrepancies and reduced booking response time from hours to minutes

✓ Empowered local Davao City business with professional e-commerce solution and real-time analytics

What I Learned

Leading this System Integration project end-to-end taught me the full software development lifecycle—from stakeholder meetings and requirements analysis to architecture design, implementation, deployment, and documentation. I gained deep understanding of how to integrate multiple business systems, the importance of comprehensive planning before coding, and how proper documentation ensures long-term maintainability. Most importantly, I learned to deliver solutions that directly solve real business problems for local entrepreneurs.

The Solution

I developed a comprehensive gym management system with QR code scanning technology that allows students to check in instantly using their School ID. The system features three distinct interfaces: student access through QR scanning, frontdesk operations dashboard, and finance management panel. This solution eliminates manual processes, reduces wait times, and provides real-time tracking of gym operations.

Key Features & Functionality

  • QR Code Scanner Integration: Students can quickly check in by scanning their School ID QR code, enabling instant membership verification and access
  • Role-Based Access Control: Three distinct user types with customized dashboards: Student Access (QR scanning), Frontdesk Operations, and Finance Management
  • Frontdesk Dashboard: Comprehensive interface for staff to manage memberships, view attendance records, handle member inquiries, and monitor real-time gym activity
  • Finance Management Panel: Dedicated interface for managing payments, membership renewals, financial reports, and revenue tracking
  • Real-Time Attendance Tracking: Automatic logging of student check-ins with timestamps and usage analytics
  • Membership Management: Complete CRUD operations for student memberships with status tracking and renewal reminders
  • Analytics Dashboard: Visual representation of gym usage, peak hours, attendance trends, and capacity management

Technical Implementation

  • QR Code Scanning: Implemented browser-based QR code scanner using JavaScript libraries for real-time student ID scanning and verification
  • Frontend Architecture: Built with semantic HTML5, custom CSS3, and vanilla JavaScript for optimal performance and maintainability
  • Bootstrap 5 Framework: Responsive design ensuring seamless experience across desktop, tablet, and mobile devices
  • LocalStorage Database: Client-side data persistence for memberships, attendance records, and system configuration without backend infrastructure
  • Role-Based Authentication: Secure login system with separate credentials for frontdesk and finance roles, with student access via QR scanning
  • Responsive Design: Mobile-first approach ensuring the scanner works perfectly on various devices including tablets used at the gym entrance

Results & Impact

✓ Reduced student check-in time from 2-3 minutes to under 5 seconds with QR code scanning

✓ Eliminated long queues during peak hours with instant membership verification

✓ Streamlined frontdesk operations with dedicated dashboard for staff efficiency

✓ Improved financial tracking with separate finance management interface

✓ Enabled real-time analytics for gym usage patterns and capacity planning

✓ Zero infrastructure costs with client-side implementation and GitHub Pages hosting

What I Learned

Building this QR code scanner system taught me the importance of user experience in high-traffic environments. Seeing students check in instantly during busy gym hours reinforced that technology should make life easier, not more complicated. I learned to design for speed and reliability—when hundreds of students need access quickly, every second counts. The project also taught me how to implement role-based systems that serve different user needs while maintaining data integrity. Most importantly, working on a system used daily by real students and staff showed me that great software is invisible when it works perfectly, and that's exactly what we achieved.
